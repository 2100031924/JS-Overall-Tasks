SECTION 1 – Array Higher Order Methods Tasks
Task 1 – forEach() Practice
let subjects = ["Maths", "Science", "English", "History"]
Requirements:
• Use forEach()
• Print Subject 1: Maths ... Subject 4: History
• Store result in a variable
• Question: Why does forEach() return undefined?

Ans:
let subjects = ["Maths", "Science", "English", "History"];
let result = subjects.forEach((subject, index) => {
    console.log(`Subject ${index + 1}: ${subject}`);
}); // result will be undefined

forEach() return undefined why?
Ans: 
By design, forEach() is meant to execute a side effect (like logging or updating a variable outside the loop) for each element. It does not return a new array or value because its purpose is iteration, not transformation.

Task 4 – find() Practice
Use same students array
Requirements:
• Use find()
• Find first student scoring above 50
• Print that object
• Question: Difference between filter() and find()?

Ans:
let firstTopper = students.find(student => student.marks > 50);
console.log(firstTopper); // { name: "B", marks: 75 }

Difference: filter() returns an array of all elements that match the condition. find() returns only the first element that matches; once it finds a match, it stops searching.






Task 5 – reduce() Practice
let cart = [
 { item: "Shirt", price: 1000 },
 { item: "Shoes", price: 2000 },
 { item: "Watch", price: 3000 }
]
Requirements:
• Use reduce()
• Calculate total price
• Bonus: Add 5% tax inside reduce()

Ans:
let cart = [
 { item: "Shirt", price: 1000 },
 { item: "Shoes", price: 2000 },
 { item: "Watch", price: 3000 }
];

let totalWithTax = cart.reduce((acc, current) => {
    let tax = current.price * 0.05;
    return acc + current.price + tax;
}, 0);
console.log(totalWithTax); // 6300

Task 8 – sort() Practice
let salaries = [50000, 10000, 70000, 30000]
Requirements:
• Sort ascending
• Sort descending
• Explain why normal sort() fails for numbers

Ans:
let salaries = [50000, 10000, 70000, 30000];

let ascending = [...salaries].sort((a, b) => a - b);
let descending = [...salaries].sort((a, b) => b - a);

Why does normal sort() fail for numbers? By default, sort() converts elements to strings and compares their UTF-16 code unit values. For example, "10000" comes before "3000" because "1" is less than "3" alphabetically. We use a compare function (a, b) => a - b to fix this.

FINAL TEAM CHALLENGE – Employee Report System
let employees = [
 { name: "Naveen", salary: 50000 },
 { name: "Arun", salary: 30000 },
 { name: "Kiran", salary: 70000 }
]
Requirements:
• Convert names to uppercase using map()
• Filter salary > 40000
• Find first salary > 60000
• Calculate total salary using reduce()
• Sort salaries Descending

Ans:
let employees = [
 { name: "Naveen", salary: 50000 },
 { name: "Arun", salary: 30000 },
 { name: "Kiran", salary: 70000 }
];

// Convert names to uppercase
let upperNames = employees.map(emp => ({
  ...emp,
  name: emp.name.toUpperCase()
}));
console.log(upperNames);

// Filter salary > 40000
let highSalary = employees.filter(emp => emp.salary > 40000);
console.log(highSalary);

// Find first salary > 60000
let firstHigh = employees.find(emp => emp.salary > 60000);
console.log(firstHigh);

// Calculate total salary
let totalSalary = employees.reduce((sum, emp) => sum + emp.salary, 0);
console.log(totalSalary);  // 150000

// Sort salaries descending
let sorted = employees.sort((a, b) => b.salary - a.salary);
console.log(sorted);
